# Internationalization (i18n)

## Language files

```
lang/
├── en/
│   ├── auth.php
│   ├── validation.php
│   └── flights.php
└── es/
    ├── auth.php
    └── flights.php
```

```php
// lang/en/flights.php
return [
    'welcome'       => 'Welcome, :name',
    'found'         => ':count flight found|:count flights found',
    'no_results'    => 'No flights available for this route.',
    'book'          => 'Book now',
];
```

## Using translations

```php
// Simple
__('flights.welcome', ['name' => 'Raúl']);
// → "Welcome, Raúl"

trans('flights.no_results');
// → "No flights available for this route."

// Pluralization
trans_choice('flights.found', 1);   // → "1 flight found"
trans_choice('flights.found', 5);   // → "5 flights found"
```

## In Twig templates

```twig
{{ __('flights.welcome', {name: user.name}) }}
{{ trans_choice('flights.found', count) }}

{# Shorthand #}
{{ 'flights.book' | trans }}
```

## Setting the locale

```php
// In a middleware or at boot
app()->setLocale('es');

// Current locale
app()->getLocale();   // 'es'
```

## Configuration

```ini
APP_LOCALE=en
APP_FALLBACK_LOCALE=en
```

```php
// config/app.php
return [
    'locale'          => env('APP_LOCALE', 'en'),
    'fallback_locale' => env('APP_FALLBACK_LOCALE', 'en'),
];
```

## Validation messages

```php
// lang/en/validation.php
return [
    'required' => 'The :attribute field is required.',
    'email'    => 'The :attribute must be a valid email address.',
    'min'      => [
        'string' => 'The :attribute must be at least :min characters.',
        'numeric' => 'The :attribute must be at least :min.',
    ],
];
```
